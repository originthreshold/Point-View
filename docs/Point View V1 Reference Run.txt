Its purpose is to preserve a concrete example that matches the current documented model, parameter defaults, and observed behavior.

This file should be used as a consistency anchor when transferring later code or UI pieces.

## Why This Run Matters

This run is one of the clearest anchor cases in the current Point View V1 project state.

It shows that the system can stop on `accumulated_drift` rather than only on origin separation or hard rupture.

That makes the behavior more plausible for gradual grayscale travel.

## What This Run Demonstrates

1. small variation is not treated as immediate rupture
2. gradual travel can build meaningful drift
3. the journey can matter, not just final distance from the origin
4. stop type is a critical part of the output
5. the current default parameter set has a concrete behavioral reference

## Consistency Rule

Future transferred code, UI readouts, or test harnesses should not silently contradict this reference run.

If later implementation differs, one of the following must happen deliberately:

- the implementation is corrected to match this anchor
- the docs are revised with a clearly stated reason
- a new reference run is added and explicitly promoted
